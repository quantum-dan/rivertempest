tempest2fast.prediction module¶

Created on Tue Mar 4 13:40:21 2025

@author: dphilippus

This file handles the actual prediction process. Namely:

  • Load spatial surfaces and coefficients

  • Retrieve all data inputs

  • Process LST and humidity to generate the required summaries

  • Estimate coefficients

  • Compute seasonality

  • Compute anomalies

  • Store result

As a matter of general procedure, summaries and coefficients will be quite compact (no timesteps) and can therefore be pre-generated and stored. The coefficients themselves may be a useful data result, since that amounts to a reasonably thorough description of thermal regimes.

Prediction, on the other hand, generates a lot of data, expected to be around 300 GB. We don’t want to have a lot of intermediate data floating around, and I think reducing storage requirements is likely a higher priority than reducing runtime. Thus, although it involves some redundant computation, each of the three primary outputs (temp.doy, temp.anom, temp.plus) is generated in a single pass (one ufunc):

  • temp.doy: (doy, coefficients) -> temp.doy

  • temp.plus and temp.anom: (doy, LST, humidity, coefficients) ->

    (lst anom, humidity anom, coefficients) -> temp.plus/temp.anom

This gives us three variables of the result dataset. The other two, then, are just sums of the relevant variables.

Testing paths:

Run all the way through coefficient estimation and timeseries generation, but with dataframes as inputs. Allows using TE2FAST as regular TempEst 2-but-in-Python. Since this version is not grid-based, it is assumed that all input coordinates are only for points of interest. See make_masked_dataset docs for detailed input info.

  • fixed (str or DataFrame) – Either a DataFrame of fixed coefficients or the path thereof.

  • inpdata (str or DataFrame) – DataFrame containing all input columns: lat, lon, date, elevation, water, shrubland, grassland, barren, LST, humidity; or path thereof.

  • spatmod (str or DataSet) – Spatial component of coefficient krigs. Still a DataSet, unlike the rest.

  • idcol (str, optional) – Name of the ID column, if any. The default is None.

Returns:

  • DataFrame (lon x lat x date, possibly idcol) containing the five predicted timeseries

  • temp.doy, temp.anom, temp.plus, temp.mod, temp.max

Notes

Just expanding the input points into a grid does not work well, since you end up with N^2 lat/lon pairs. Instead, we build a <space x time> (2D) grid where the space index is assigned from lat/lon, and include lat/lon as data.

tempest2fast.prediction.make_masked_dataset(rivers: str | DataArray, dem: str | DataArray, lcov: str | Dataset, lst: str | DataArray, humidity: str | DataArray, spatmod: str | Dataset, bbox: tuple[float, float, float, float] = None)¶

Generate a combined dataset of all required inputs, masked to the river network. All inputs can be either a string (the path to the file[s]) or an xarray DataSet/DataArray. Strings are assumed to be the full path for static datasets (rivers, dem, lcov) and to contain a wildcard for a multifile dataset (separated by date) for the dynamic datasets (lst, humidity).

All datasets should be on the 0.01-degree grid. Coordinates should be lat, lon, and (if applicable) date.

Everything except lcov and spatmodel should be a DataArray, if provided pre-loaded. lcov should be a dataset with variables “water”, “shrubland”, “grassland”, and “barren” (all abundances). spatmodel should be a dataset with variables corresponding to each coefficient.

Parameters:

  • rivers (str or DataArray) – True/False array indicating the presence of a river in the pixel.

  • dem (str or DataArray) – Elevations in meters.

  • lcov (str or Dataset) – Land cover layers as fractional abundances (water, shrubland, grassland, barren).

  • lst (str or DataArray) – Land surface temperature in Kelvin.

  • humidity (str or DataArray) – Specific humidity in kg/kg.

  • spatmod (str or Dataset) – Spatial component of coefficient krigs.

  • bbox (optional (float, float, float, float)) – (left, bottom, right, top). The default is None. If provided, clip to the specified bounding box. Only applied if inputs are paths, not datasets.

Returns:

Two datasets

Return type:

static and dynamic, masked to the rivers.

Notes

If loading from files, FAST makes some format assumptions based on the default processing procedures. This may be different if you generate the files yourself.

Even with Xarray’s lazy loading - but much improved thereby - this does have substantial memory requirements. In a 1-year test, the input size on disk is about 180 MB/day (LST + humidity), for a total size of ~66 GB. Memory usage when processing hits about 5.5 GB. With a 10-degree-square bounding box, that’s down to 2.5 GB. Static is 124 MB and dynamic is 4 GB.

tempest2fast.rivers module¶

Created on Tue Mar 4 13:39:57 2025

@author: dphilippus

This file handles river masking for the TE2FAST grid. What and why?

What: in each 0.01-degree grid cell, identify whether there is a river. Provide a gridded dataset of rivers that can also be used for masking.

Why:

  • For user convenience: analysis can be limited to actual rivers - no

mountaintop guesses. - For processing and storage efficiency: we don’t need to analyze non-river sites.

Process the MERIT Hydro river width dataset, downloaded using Google Earth Engine as RiverWidth.tif. For every 0.01-degree cell, note whether it has nonzero mean river width (i.e., contains a river). In the original projection, 5% of cells contain a river.
